Руководство по Crypto Wallet

Crypto Wallet - это приложение, которое создано для хранения и перевода CRU/UNTB/USDU токенов, выпущенных на смарт-контракте EOS  для CryptoUnit  с дальнейшей возможностью управления ими, отображением полученных дивидендов от стейкинга и иными действиями в рамках реализуемой логики.

Генерация ключа

Для зарегистрированных в Личном кабинете пользователей и прошедших верификацию доступна возможность генерации секретной фразы (Seed Phrase), который в дальнейшем будет необходим для авторизации пользователя.

Генерация секретного ключа происходит из Личного кабинета пользователя путем нажатия на соответствующую кнопку, после чего осуществляется переход на экран Выбора способа хранения сгенерированного секретного ключа.

На данном экране пользователь видит свою Seed Phrase. А также может выбрать метод хранения Seed Phrase: Basic или Advanced. 

Basic - всю ответственность за хранение и восстановление ключа несет компания и в случае утери приватного ключа, компания сможет сформировать новый ключ доступа.

Advanced предполагает, что пользователь полностью берет на себя ответственность  за хранение ключа и последствия, связанные с его утерей.

Если пользователь выбирает Basic, нажимает Continue, переходит к следующему экрану, позволяющему ввести почту/подтвердить в поле Enter your Email ,нажать кнопку Proceed, которая будет использоваться для дальнейшего восстановления ключа. В дальнейшем email будет использоваться для восстановления приватного ключа пользователя.

Пользователю на почту приходит код авторизации.

Пользователь должен ввести в поле шестизначный код, который ему пришел на почту и нажать Done.

Авторизация

Авторизация пользователя  предполагает вход по секретному ключу (Private Key), который в дальнейшем  хранится в локальном хранилище браузера. В дальнейшем он будет требоваться для подтверждения совершения транзакций.

Для успешной авторизации в приложении пользователь должен ввести  секретный ключ доступа. Экран авторизации:

После ввода секретного ключа доступа нужно нажать Sign in для продолжения :

При успешной авторизации при первом входе пользователю необходимо ввести 6-значный ПИН-код, во втором поле для ввода его подтвердить и нажать Done.

 Восстановление ключа

В случае, если пользователь при Генерации ключей выбрал способ хранения Basic, тем самым возложив ответственность по хранению и восстановлению на компании, в случае утери ключа на экране авторизации он может нажать на ссылку I’ve lost my Private Key.

Пользователю в открывшемся поп-апе необходимо будет вписать email, на который придет ссылка для последующего восстановления ключа и нажать на кнопку “Send message to this email”.

Об успешной отправке email будет свидетельствовать соответствующее сообщение.

Пользователь переходит по ссылке, отправленной в письме, и попадает на экран с отображением нового сгенерированного ключа, который необходимо сохранить для дальнейшего входа в кошелек.

Дашборд

После ввода 6-значного пин-кода и его подтверждения, пользователь переходит на главный экран кошелька.

 Главный экран состоит из модулей, которые содержат:

  • информацию о пользователе и хэдр с вкладками;
  • информацию о состоянии баланса пользователя;
  • историю его транзакций.

Блок “Информация о пользователе” содержит следующую информацию:

  • имя пользователя;
  • дата создания аккаунта;
  • знак копирования username.

Сверху Главного экрана находится информация о вычислительных мощностях:

  • три прогресс бара, отображающих информацию о CPU, NET и RAM (процессор, пропускная способность сети и оперативная память)
  • информация, сколько токенов застейкано для CRU, NET.

Под каждым прогресс баром есть кнопки Staked/Unstaked и для RAM Buy/Sell RAM, нажав на которые пользователь может купить или продать оперативную память, или застейкать токены соответственно.

Также под прогресс барами находятся кнопки, нажав на которые пользователь сможет перейти к определенным страницам:

  • Wallet Dashboard
  • Convert WCRU
  • Sent
  • CRU Staking
  • WCRU Staking
  • UNTB Staking
  • Buy/Sell RAM
  • Vote
  • Transactions history

Блок “Мое портфолио” содержит информацию о балансах пользователя:

  • итоговый баланс в USDU;
  • CRU со статусами Staked/Unstaked/Frozen;
  • WCRU со статусами Staked/Unstaked/Frozen;
  • UNTB со статусами Staked/Staked.

  Сверху портфолио находится кнопка Refresh balances, нажав на которую пользователь может обновить информацию по своему балансу.

Блок “Последние транзакции” отображает последнюю историю транзакций  пользователя. При нажатии в списке транзакций на транзакцию откроется экран с детальным ее описанием, который содержит следующее:

  • номер блока;
  • время создания блока;
  • ID транзакции;
  • количество действий;
  • username подписавшего транзакцию

На странице Transactions history также находится кнопка Refresh, при нажатии на которую пользователь может обновить информацию по истории транзакций, а также может выставить следующие фильтры:

Filter by action:

  • All Actions
  • Send tokens
  • Staking
  • RAM/CRU/NET
  • Votes
  • Account bids
  • Claim Reward

Filter by asset:

  • All
  • UNTB
  • CRU
  • USDU

Отправка токенов

Экран перевода токенов дает возможность пользователю переводить токены с одного аккаунта на другой. Содержит следующие поля:

  • позволяет выбрать вид токена для перевода (выпадающий список);
  • количество свободных токенов;
  • получатель (account name);
  • Количество  токенов;
  • Memo (примечание)
  • кнопка Отправить
  • ввод ПИН-кода для отправки
  • Сумма для отправки
  • Отправить все send all

Чтобы перевести токены другому пользователю или на другой кошелек пользователя, нужно в поле You send ввести количество токенов, выбрать вид токенов. В поле Send to ввести имя пользователя и нажать кнопку Send.

Также на странице Sent можно посмотреть историю транзакций.

Стейкинг токена CRU

Чтобы застейкать токен CRU, в верхней вкладке выбираем CRU Staking, переходим на следующий экран:

Результатом стейкинга токенов CRU является начисление токенов UNTB на аккаунт пользователя.

В поле Stake CRU вводим количество токенов CRU и нажимаем кнопку Stake CRU.

После того, как застейкали, available balance уменьшается, staked balance увеличивается на количество застейканых токенов.

Также можно проводить обратную операцию. В поле Unstake CRU пользователь вводит количество токенов CRU и нажимает кнопку Unstake CRU. Балансы меняются в противоположную сторону на количество переведенных токенов CRU.

При стейкинге токинов CRU накапливаются вознаграждения в UNTB. Пользователь может забрать в любое время вознаграждения, нажав на кнопку Claim UNTB.

Переведенное количество токенов UNTB отображается на главном экране кошелька.

Также внизу страницы CRU staking можно посмотреть историю транзакций.

Стейкинг токена UNTB

Стейкинг предполагает механизм блокировки токенов на определенное время через блокировку CPU и NET.

Страница содержит вкладки Stake/Unstake.

Вкладка Stake представлена следующими полями:

  • количество  CPU для стейка (в EOS)
  • поле для ввода информации
  • выбор в % выражении из 4 величин (25%, 50%, 75%, 100%)
  • количество  NET для стейка (в EOS)
  • поле для ввода информации
  • выбор в % выражении из 4 величин (25%, 50%, 75%, 100%)

Вкладка Unstake представлена следующими полями:

  • количество  CPU для unstake (в EOS)
  • поле для ввода информации
  • количество  NET для unstake (в EOS)
  • поле для ввода информации

В поле Stake UNTB вводим количество токенов UNTB и нажимаем кнопку Stake UNTB.

После того, как застейкали, available balance уменьшается, staked balance увеличивается на количество застейканых токенов.

Также можно проводить обратную операцию. В поле Unstake UNTB пользователь вводит количество токенов UNTB и нажимает кнопку Unstake UNTB. Балансы меняются в противоположную сторону на количество переведенных токенов UNTB.

Пользователь также может не вводить токены вручную, а выбрать процентное соотношение ( 25 %, 50 , 75 и 100) от общего баланса.

Продажа/покупка RAM (оперативной памяти)

Покупка RAM предполагает покупку пользователем оперативной памяти по рыночной цене EOS.

Определенный тип данных предполагает наличие у пользователя RAM для хранения данных на блокчейне. Например, оперативная память нужна при создании аккаунта, константы из смарт-контракта, баланса, чтобы внести запись о нем.

Страница содержит вкладки Купить/Продать.

Вкладка Купить представлена следующими полями:

  • получатель RAM  (никнейм пользователя)
  • радиобатон с выбором отображения эквивалента покупки RAM
  • байт
  • токен
  • количество  RAM  для покупки (отображается в выбранном эквиваленте)

Вкладка Продать представлена следующими полями:

  • количество  RAM пользователя (отображается в величинах байт и токен):
  • куплено
  • используется
  • доступно
  • количество  RAM для продажи (отображается в величинах байт и токен)
  • поле для ввода информации

Чтобы купить оперативную память, пользователь в поле Buy RAM должен ввести количество токенов и нажать кнопку Buy RAM.

Чтобы продать оперативную память, пользователь в поле Sell RAM должен ввести количество

Голосование

Экран Голосование за валидаторов  дает возможность пользователю голосовать застейкаными UNTB за лучшего производителя блоков.  Каждый пользователь может проголосовать не более чем за 30 кандидатов.

По результатам голосования лучшие 21 производителя блоков становятся “основными” производителями.

Если у пользователя застейкано 10 UNTB, то он может голосовать 10 UNTB за каждого из кандидатов.

Таблица со списком производителей блоков для голосования содержит следующие поля:

  • позиция кандидата;
  • никнейм;
  • статус;
  • местоположение (страна);
  • соотношение голосов;
  • общее количество голосов;
  • количество заработанных токенов EOS;
  • Чекбокс для выбора кандидата

Чтобы проголосовать, пользователь может в чекбокс поставить галку как за одного кандидата, так и за нескольких одновременно и нажать кнопку Vote to Producers.
